# ShopFlow — Microservices E-Commerce Platform

A multi-service e-commerce backend designed to practice real-world DevOps and Cloud Engineering workflows.

> **How this project works:** The application code (5 microservices) and Linux break-fix troubleshooting scripts were generated by [Claude](https://claude.ai) to simulate a realistic production environment. The DevOps infrastructure — Dockerfiles, Docker Compose, Terraform, and deployment configuration — was written by me as hands-on practice.

## Services

| Service | Language | Port | Description |
|---------|----------|------|-------------|
| **Auth Service** | Node.js / Express | 3001 | User registration, JWT authentication, role-based access control |
| **Product Service** | Python / Flask | 3002 | Product catalog CRUD, category management, stock tracking with low-stock alerts |
| **Order Service** | Node.js / Express | 3003 | Order processing, status lifecycle management, inter-service stock validation |
| **Notification Worker** | Python | 3004 | Background worker consuming Redis pub/sub events (orders, stock alerts) |
| **Frontend** | Static HTML/JS/CSS | 8080 | Admin dashboard with login, product/order views, and service health monitoring |

## Infrastructure (What I Built)

### Docker

- **Dockerfiles** for all 5 services (Node.js Alpine, Python Slim, nginx Alpine)
- **Docker Compose** orchestrating the full stack with PostgreSQL 15, Redis 7, health checks, and proper service dependency ordering
- **init.sql** to bootstrap the three separate databases required by the services

### Terraform (AWS)

Infrastructure as Code provisioning on AWS (`ap-south-1`):

- Custom **VPC** with public subnet, internet gateway, and route table
- **Security group** with SSH access and IP-scoped frontend access
- **EC2 instance** (t3.medium) with auto-generated SSH key pair
- Outputs the public IP for easy SSH access

## Break-Fix Scenarios (Linux Troubleshooting)

The `break-fix/` directory contains 8 simulated production incidents with progressive difficulty. Each scenario includes:

- **`break.sh`** — Introduces the problem (run as root)
- **`ticket.md`** — A realistic support ticket describing symptoms
- **`verify.sh`** — Validates if the fix was applied correctly

| # | Scenario | Difficulty | Skills |
|---|----------|:----------:|--------|
| 01 | File Permissions | ⭐ | chmod, chown, ls -la |
| 02 | Port Conflict | ⭐ | ss, lsof, ps, kill |
| 03 | Database Connectivity | ⭐⭐ | pg_hba.conf, postgresql.conf, systemctl |
| 04 | Disk Full | ⭐⭐ | df, du, find, hidden files |
| 05 | Inter-Service Networking | ⭐⭐ | /etc/hosts, env vars, curl |
| 06 | Firewall Rules | ⭐⭐⭐ | iptables, ufw, network debugging |
| 07 | Runaway Processes | ⭐⭐⭐ | top, ps, kill, free, renice |
| 08 | Systemd Crash Loop | ⭐⭐⭐ | systemctl, journalctl, service files |

## Quick Start

```bash
# Clone and start everything
git clone https://github.com/<your-username>/shopflow.git
cd shopflow
docker compose up --build

# Services will be available at:
# Frontend:     http://localhost:8080
# Auth API:     http://localhost:3001/api/health
# Product API:  http://localhost:3002/api/health
# Order API:    http://localhost:3003/api/health
```

**Default login:** `admin@shopflow.io` / `admin12345`
